# 프로젝트 컨벤션 — JP Product Trend Dashboard

이 저장소에서 작업하는 Claude Code 인스턴스용 룰.
(repo 디렉토리명은 `competitor_new_products` 유지 — git 경로 호환)

---

## 🎯 목적

상품기획자가 매주 받아볼 **타사(경쟁사) 신상품 리포트**의 자동 수집·정리·시각화.

- 수집 대상: Qoo10.jp 한국 K-뷰티 경쟁사 + 일부 뷰티 디바이스 셀러
- 출력 양식: 구글 시트 "타사 신제품" 컬럼 미러 (스킨/색조/ETC)
- 사용자: 상품기획팀 (의사결정용 — 우리 신제품 라인업 기획에 활용)

---

## 🔑 [필수] 대시보드 UI에는 "수치 + 해석" 함께 표시

메인 ad_dash 룰과 동일. 모든 카드/메트릭/상품 행에:
- ❌ 금지: 가격·이미지만 단독 노출
- ✅ 필수: **무엇이 새로운가** + **왜 주목해야 하는가** + **우리는 어떻게 활용할까**

예시 패턴:
- `medicube 아젤라인산 토너 ¥2,350 — 트러블·진정 신규. 자사 토너 카테고리 비교군 등록`
- `rom&nd 신규 색상 8종 추가 — Q2 메가와리 노림. 자사 컬러 컨피그 점검 필요`

---

## 📦 데이터 흐름

```
Qoo10 mobile shop pages
    ↓ (crawler/qoo10_new_products.py — 매일 09:00 JST)
data/_snapshots/{date}.json   ← 일자별 raw
    ↓ (rollup)
data/new_products.json        ← 통합 (직전 30일)
    ↓ (GitHub Actions copy)
dashboard/data/new_products.json
    ↓ (Cloudflare Pages 정적)
브라우저
```

## 🚫 멀티 인스턴스 충돌 방지

- 이 프로젝트는 **메인 ad_dash (`Downloads/ad_dash_v5.9.15`)와 완전 분리**.
- `crawler/_brands.py`는 메인의 그것을 카피해서 시작하되, 이후 양쪽 독립 관리.
- 메인 프로젝트의 데이터/HTML 직접 참조 금지 (별도 git 원칙).

## 📐 자사 4브랜드 (참고용 — 비교군 표시할 때만)

- BOH (바이오힐보) / 브링그린 — 기초
- 웨이크메이크 / 컬러그램 — 색조

자사 신제품은 직접 알고 있으니 크롤 대상에 포함하지 않음. 단, 대시보드에서 "우리 라인업 대비 비교" 시 라벨링 용도로만 사용.

## 🔄 신상품 정의 (확정)

1. Qoo10 상품명 또는 카드에 NEW / 新発売 / 新着 마커
2. 또는 셀러 페이지 "新着順" 정렬 시 상위 N (등록일 ≤ 30일 추정)
3. 또는 상품 상세 페이지에서 등록일 ≤ 오늘 - 30일

세 조건 중 하나라도 만족하면 candidate. 상세페이지 fetch로 정확한 등록일 확보 후 30일 컷.
